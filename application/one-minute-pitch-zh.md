# 一分钟申请陈述

如果申请表有“补充说明”或需要简短介绍，可以使用这一版：

我正在搭建一个面向个人开发者和小团队的 AI Coding Agent 工作流，目标是用 MiMo-V2.5-Pro 的长上下文能力完成代码理解、需求拆解、代码生成、Review、测试修复和文档生成。

这个项目非常适合 MiMo，因为真实 coding-agent 任务通常需要一次性读取完整代码仓库、需求文档、测试日志和多轮修改记录。MiMo 的 100 万上下文窗口可以减少上下文切片和信息丢失，提高 Agent 在复杂工程任务中的稳定性。

我申请 Max Plan 是因为项目需要连续运行真实任务评估，而不是一次性体验。单个任务预计消耗 10 万到 100 万 Tokens，每日评估预计消耗 300 万到 800 万 Tokens。高额度将用于不少于 20 个 coding-agent 任务测试、多模型对比、Prompt 模板优化和实践文档整理。

如果获得额度，我会将工作流、Prompt 模板、评估记录和 MiMo Coding Agent 使用经验整理为 GitHub 示例仓库或公开实践文章，为 MiMo 生态提供真实开发者反馈和可复用案例。
